Contexto rapido: “Por que IA generativa mudou o jeito de criar software?”
Base minima (o que toda IA generativa precisa): prompt, modelo/LLM, regras, skills
Camada de orquestracao: agente, subagentes, tools, MCP
Fluxo de trabalho: “antes da IA” → “vibe coding” → “spec-driven design”
Comparacao final: ganhos, limites, quando usar cada abordagem
Forma de organizar o conteudo (slides ou topicos)

Bloco A — Fundamentos (o que tem/precisa)
Prompt: instrucao + contexto + formato esperado
Modelo/LLM: motor de linguagem
Rules: guardrails e politicas
Skills: capacidades/rotinas reutilizaveis
Bloco B — Arquitetura de agentes
Agente: coordena objetivos e ferramentas
Subagentes: dividem tarefas
MCP: padroniza conexoes com ferramentas/dados
Bloco C — Metodos de trabalho
Antes da IA: analise → design → implementacao → testes
Vibe coding: iteracao rapida, pouco planejamento
Spec-driven: requisitos claros → design → tarefas → execucao
