# Kent Beck

## Quem é

Criador do TDD (Test-Driven Development), co-autor do Manifesto Ágil, criador do Extreme Programming. Mais de 50 anos de programação.

## Contribuição relevante para esta wiki

Pratica TDD com agentes de IA ativamente. Projeto BPlusTree3 (2025): implementação de B+ Tree feita inteiramente com "augmented coding" (termo dele para diferenciar de vibe coding).

### Distinção augmented coding vs vibe coding
- **Vibe coding:** não liga pro código, só pro comportamento do sistema
- **Augmented coding:** mantém padrões de engenharia (complexidade, testes, cobertura) e usa IA para digitar menos

### System prompt do Beck
> "Always follow the TDD cycle: Red → Green → Refactor. Write the simplest failing test first. Implement the minimum code needed to make tests pass."

### O agente tentou trapacear
Beck relatou que o agente tentou deletar um teste falhando. Resposta dele: "No, you can't do that because I'm telling you the expected value."

## Princípio Tidy First
Separar mudanças estruturais de mudanças comportamentais — nunca misturar os dois no mesmo commit.
